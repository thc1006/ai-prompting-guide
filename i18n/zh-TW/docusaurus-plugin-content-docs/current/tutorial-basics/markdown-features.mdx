# Markdown 功能

Docusaurus 支援 **[Markdown](https://daringfireball.net/projects/markdown/syntax)** 以及一些**額外功能**。

## Front Matter

Markdown 文件的頂部有一個被稱為 [Front Matter](https://jekyllrb.com/docs/front-matter/) 的元數據區塊：

```text title="my-doc.md"
---
id: my-doc-id
title: My document title
description: My document description
slug: /my-custom-url
---

## Markdown 標題

帶有[連結](./hello.md)的 Markdown 文字
```

## 連結

支援一般的 Markdown 連結，可以使用 URL 路徑或相對檔案路徑。

```md
讓我將你導向[入門教學頁面](../intro)。
```

讓我將你導向[入門教學頁面](../intro)。

## 圖片

支援一般的 Markdown 圖片。

你可以使用絕對路徑來參照 static 目錄中的圖片（`static/img/logo.svg`）：

```md
![Docusaurus logo](/img/logo.svg)
```

![Docusaurus logo](/img/logo.svg)

你也可以使用相對於當前檔案的路徑來參照圖片。這對於將圖片放置在使用它們的 Markdown 檔案附近特別有用：

```md
![Docusaurus logo](/img/logo.svg)
```

## 程式碼區塊

支援帶有語法高亮的 Markdown 程式碼區塊。

    ```jsx title="src/components/HelloDocusaurus.js"
    function HelloDocusaurus() {
        return (
            <h1>Hello, Docusaurus!</h1>
        )
    }
    ```

```jsx title="src/components/HelloDocusaurus.js"
function HelloDocusaurus() {
  return <h1>Hello, Docusaurus!</h1>;
}
```

## 警告方塊

Docusaurus 有特殊的語法來建立警告和提示方塊：

    :::tip 我的小訣竅

    使用這個強大的功能選項

    :::

    :::danger 要小心

    這個操作很危險

    :::

:::tip 我的小訣竅

使用這個強大的功能選項

:::

:::

## MDX 和 React 組件

[MDX](https://mdxjs.com/) 可以讓你的文件更加**互動**，並允許在 Markdown 中使用任何 **React 組件**：

```jsx
export const Highlight = ({children, color}) => (
  <span
    style={{
      backgroundColor: color,
      borderRadius: '20px',
      color: '#fff',
      padding: '10px',
      cursor: 'pointer',
    }}
    onClick={() => {
      alert(`你點擊了顏色 ${color}，標籤為 ${children}`)
    }}>
    {children}
  </span>
);

這是 <Highlight color="#25c2a0">Docusaurus 綠色</Highlight>！

這是 <Highlight color="#1877F2">Facebook 藍色</Highlight>！
```
